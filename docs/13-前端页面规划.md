# 前端页面规划

## 1. 页面定位

CampusOps 前端采用“管理后台 + 工单工作台”的产品形态。

第一阶段前端目标不是做营销页或炫酷大屏，而是让不同角色能够高效完成工单提交、分派、处理、确认、知识查询和运营管理。

## 2. 技术栈约束

前端第一阶段使用：

- Vue 3
- Vite
- JavaScript
- Vue Router
- Pinia
- Element Plus
- Axios
- ECharts

第一阶段不使用 TypeScript。

当前前端开发端口固定为：

```text
3017
```

前端请求统一以 `/campusops` 作为 `baseURL`，例如登录接口在前端封装为 `/auth/login`，实际请求路径为 `/campusops/auth/login`。

## 3. 布局结构

登录后主界面采用后台管理布局：

- 左侧菜单栏。
- 顶部用户信息和快捷操作。
- 中间主内容区域。
- 工单详情页可增加右侧 AI 辅助面板。

建议一级菜单：

- 首页
- 工单中心
- 知识库
- 统计分析
- 服务管理
- 系统管理
- AI 助手

菜单应根据用户角色动态展示。

## 4. 页面清单

### 4.1 登录页

路径建议：

```text
/login
```

主要功能：

- 用户名密码登录。
- 登录失败提示。
- 登录成功后保存 Token 和当前用户信息。
- 根据角色进入首页。

对应接口：

```text
POST /campusops/auth/login
GET /campusops/auth/me
```

### 4.2 首页仪表盘

路径建议：

```text
/dashboard
```

主要功能：

- 展示当前用户相关工单概览。
- 展示待处理、处理中、已完成、超时数量。
- 服务管理员可查看全局统计。
- 普通用户只看自己的工单概览。

### 4.3 我的工单

路径建议：

```text
/tickets/my
```

面向角色：普通用户。

主要功能：

- 查看自己提交的工单。
- 按状态、分类、关键词筛选。
- 创建新工单。
- 查看工单详情。
- 取消未处理工单。
- 对已解决工单确认关闭和评价。

### 4.4 待处理工单

路径建议：

```text
/tickets/assigned
```

面向角色：IT 处理人。

主要功能：

- 查看分配给自己的工单。
- 按状态、优先级、分类筛选。
- 接单。
- 进入工单详情处理。

### 4.5 工单管理

路径建议：

```text
/tickets/manage
```

面向角色：服务管理员。

主要功能：

- 查看全部工单。
- 分派工单。
- 转派工单。
- 调整分类和优先级。
- 查看超时工单。
- 驳回无效工单。

### 4.6 工单详情页

路径建议：

```text
/tickets/:id
```

面向角色：普通用户、IT 处理人、服务管理员。

主要区域：

- 工单基础信息。
- 状态和优先级。
- 提交人和处理人。
- 问题描述和附件。
- 处理记录时间线。
- 评论区。
- 状态操作按钮。
- 右侧 AI 辅助面板。

AI 辅助面板第一版展示：

- AI 摘要。
- 分类建议。
- 关键词。
- 相似工单推荐。
- 知识库文章推荐。

AI 面板只展示建议，不直接修改工单状态。

### 4.7 新建工单页

路径建议：

```text
/tickets/create
```

面向角色：普通用户。

主要功能：

- 填写标题。
- 填写问题描述。
- 选择分类。
- 选择优先级。
- 上传附件。
- 提交工单。

第一版可先不做复杂富文本编辑器。

### 4.8 知识库列表页

路径建议：

```text
/knowledge
```

主要功能：

- 查看已发布知识库文章。
- 按分类筛选。
- 关键词搜索。
- 查看文章详情。

### 4.9 知识库管理页

路径建议：

```text
/knowledge/manage
```

面向角色：IT 处理人、服务管理员。

主要功能：

- 创建知识库草稿。
- 编辑文章。
- 服务管理员审核发布。
- 下线文章。
- 从已解决工单生成知识库草稿。

### 4.10 统计分析页

路径建议：

```text
/statistics
```

面向角色：服务管理员。

主要功能：

- 工单总数统计。
- 分类占比。
- 状态分布。
- 平均处理时长。
- 超时数量。
- 处理人工作量。

图表使用 ECharts。

### 4.11 服务分类管理页

路径建议：

```text
/service/categories
```

面向角色：服务管理员。

主要功能：

- 查看工单分类。
- 新增分类。
- 编辑分类。
- 启用或禁用分类。

### 4.12 用户管理页

路径建议：

```text
/admin/users
```

面向角色：系统管理员。

主要功能：

- 查看用户列表。
- 新增用户。
- 编辑用户信息。
- 禁用用户。
- 分配角色。

### 4.13 角色管理页

路径建议：

```text
/admin/roles
```

面向角色：系统管理员。

主要功能：

- 查看角色列表。
- 查看角色说明。
- 后续扩展角色权限配置。

第一版可以只实现角色查看和用户角色分配。

### 4.14 AI 调用记录页

路径建议：

```text
/ai/records
```

面向角色：服务管理员、系统管理员。

主要功能：

- 查看 AI 调用记录。
- 查看调用场景。
- 查看调用状态。
- 查看耗时。
- 查看失败原因。

第一版可以先展示 Mock AI 记录。

## 5. 第一版页面优先级

### 必做页面

- 登录页。
- 首页仪表盘。
- 我的工单。
- 新建工单页。
- 待处理工单。
- 工单管理。
- 工单详情页。
- 知识库列表页。

### 次优先页面

- 知识库管理页。
- 统计分析页。
- 服务分类管理页。
- 用户管理页。

### 可后置页面

- 角色管理页。
- AI 调用记录页。
- 复杂系统配置页。

## 6. 前端实现原则

- 页面服务业务流程，不做无关展示。
- 表格、表单、详情和状态标签优先。
- 请求统一封装在 `src/api/`。
- 路由统一配置在 `src/router/`。
- 用户和 Token 状态放在 Pinia。
- 枚举值统一放在 `src/constants/`。
- 不在页面组件里散写接口路径。
- AI 辅助面板只展示建议，关键操作必须由用户点击确认。

## 7. 当前工程骨架

当前已建立的前端基础结构：

```text
campusops-front/src/
  api/
    request.js
    auth.js
    ticket.js
  layouts/
    BasicLayout.vue
  router/
    index.js
  stores/
    user.js
  views/
    LoginView.vue
    DashboardView.vue
    ai/
    knowledge/
    ticket/
```

当前已落地页面入口：

- `/login`：登录页占位。
- `/dashboard`：工作台占位。
- `/tickets`：工单列表占位。
- `/knowledge`：知识库列表占位。
- `/ai-analysis`：AI 分析记录占位。

这些页面当前只作为前端骨架，真实业务数据需要后续接入后端接口。
