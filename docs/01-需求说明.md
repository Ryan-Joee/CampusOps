# 需求说明

## 1. 需求目标

本系统用于支撑校园 IT 服务问题的统一受理、分派、处理、跟踪、归档和知识沉淀。

系统应优先保证传统业务流程完整可用，在此基础上逐步引入 AI 辅助能力，提高工单处理效率和历史经验复用能力。

## 2. 用户需求

### 2.1 普通用户

普通用户主要是学生、教师或校内系统使用者。

普通用户需要：

- 提交 IT 服务问题。
- 上传问题截图或附件。
- 查看自己提交的工单列表。
- 查看工单处理进度。
- 补充问题描述。
- 查看处理人回复。
- 确认问题已解决。
- 对服务结果进行评价。
- 在知识库中搜索常见问题。

### 2.2 IT 处理人

IT 处理人主要是网络管理员、系统管理员、机房维护人员或技术支持人员。

IT 处理人需要：

- 查看分配给自己的工单。
- 接收或拒绝处理任务。
- 查看工单详情和用户补充信息。
- 添加处理记录。
- 回复用户。
- 修改工单处理状态。
- 标记工单已解决。
- 查看 AI 推荐的相似工单和知识库文章。
- 将典型工单沉淀为知识库草稿。

### 2.3 服务管理员

服务管理员主要负责工单调度和服务运营。

服务管理员需要：

- 查看全部工单。
- 按分类、状态、优先级、处理人、时间筛选工单。
- 设置工单分类和优先级。
- 分派或转派工单。
- 查看超时工单。
- 管理服务分类。
- 管理处理团队。
- 审核知识库文章。
- 查看服务统计数据。

### 2.4 系统管理员

系统管理员主要负责平台配置和系统维护。

系统管理员需要：

- 管理用户。
- 管理角色。
- 配置权限。
- 查看操作日志。
- 查看 AI 调用记录。
- 配置系统参数。
- 管理字典数据。

## 3. 功能需求

### 3.1 工单管理

- 创建工单。
- 查询工单列表。
- 查看工单详情。
- 编辑工单基础信息。
- 设置工单分类。
- 设置工单优先级。
- 分派处理人。
- 转派处理人。
- 变更工单状态。
- 添加处理记录。
- 添加用户评论。
- 上传和查看附件。
- 用户确认关闭工单。

### 3.2 用户与权限

- 用户登录。
- 用户信息管理。
- 角色管理。
- 接口权限控制。
- 数据权限控制。
- 不同角色看到不同的菜单和数据范围。

### 3.3 知识库

- 创建知识库文章。
- 编辑知识库文章。
- 发布或下线知识库文章。
- 按分类和关键词查询知识库。
- 将知识库文章关联到工单。
- 从已解决工单生成知识库草稿。

### 3.4 AI 辅助

第一版 AI 能力包括：

- 工单自动分类建议。
- 工单内容摘要。
- 工单关键词提取。
- 相似历史工单推荐。
- 相关知识库文章推荐。

后续可扩展：

- 回复草稿生成。
- 周报和月报生成。
- 基于知识库的问答。
- Agent 辅助诊断。
- AI 工具调用。

### 3.5 统计分析

- 工单总数统计。
- 不同状态工单数量统计。
- 不同分类工单数量统计。
- 超时工单数量统计。
- 平均处理时长统计。
- 处理人工作量统计。

### 3.6 日志审计

- 记录工单创建、分派、转派、状态变更等操作。
- 记录用户登录和重要配置变更。
- 记录 AI 调用请求、响应摘要、耗时和结果状态。

## 4. 非功能需求

- 系统应具备清晰的模块边界。
- 系统应支持基本权限隔离。
- 核心工单流程应具备自动化测试。
- AI 调用失败时不影响核心工单流程。
- 关键业务操作应记录审计日志。
- 接口应具备统一响应格式和错误处理。
- 数据库表结构应具备可演进能力。

## 5. MVP 范围

MVP 必须完成：

- 登录与角色区分。
- 工单创建、分派、处理、关闭的主流程。
- 工单列表和详情。
- 处理记录和评论。
- 基础知识库。
- 基础 AI 分类和摘要。
- 基础操作日志。

MVP 可以暂缓：

- 复杂统计报表。
- 多 Agent 协作。
- MCP 接入。
- 完整 RAG 评测体系。
- 复杂消息通知。
