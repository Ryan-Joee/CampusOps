# 数据库设计

## 1. 设计目标

数据库设计优先支撑 CampusOps 第一版核心业务闭环：用户提交工单、管理员分派、处理人处理、用户确认关闭、知识库沉淀、AI 辅助分析和操作审计。

设计原则：

- 先保证核心流程清晰稳定，再为后续扩展预留空间。
- 状态、类型、优先级等字段使用枚举编码，必要时可通过字典表扩展。
- 关键业务操作保留审计记录。
- AI 结果作为辅助分析记录保存，不直接覆盖正式业务字段。
- 所有表建议包含创建时间、更新时间和逻辑删除字段。

## 2. 命名约定

- 表名使用小写下划线风格。
- 主键统一使用 `id`。
- 外键字段使用 `{业务对象}_id`。
- 时间字段使用 `created_at`、`updated_at`。
- 逻辑删除字段使用 `deleted`。
- 状态字段使用字符串编码或短枚举编码，避免直接使用中文。

### 3.1 sys_user

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| username | varchar(64) | 登录名，唯一 |
| password_hash | varchar(255) | 密码哈希 |
| real_name | varchar(64) | 真实姓名 |
| email | varchar(128) | 邮箱，唯一，必填 |
| phone | varchar(32) | 手机号，唯一，必填 |
| department | varchar(128) | 院系或部门 |
| status | varchar(32) | 用户状态：enabled、disabled |
| last_login_at | datetime | 最近登录时间 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| deleted | boolean | 是否删除 |

### 3.2 sys_role

角色表。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| role_code | varchar(64) | 角色编码，唯一 |
| role_name | varchar(64) | 角色名称 |
| description | varchar(255) | 角色说明 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

- normal_user：普通用户

## 4. 工单核心表

### 4.1 ticket

工单主表。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| ticket_no | varchar(64) | 工单编号，唯一 |
| title | varchar(200) | 工单标题 |
| description | text | 问题描述 |
| category_id | bigint | 工单分类 ID |
| priority | varchar(32) | 优先级：low、medium、high、urgent |
| status | varchar(32) | 状态 |
| submitter_id | bigint | 提交人 ID |
| assignee_id | bigint | 当前处理人 ID，可为空 |
| assigned_at | datetime | 分派时间 |
| resolved_at | datetime | 标记解决时间 |
| closed_at | datetime | 关闭时间 |
| due_at | datetime | 期望处理截止时间 |
| source | varchar(32) | 来源：web、mobile、admin、ai_draft |
| ai_summary | varchar(1000) | 当前 AI 摘要，可为空 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| deleted | boolean | 是否删除 |

状态建议：

- pending_assignment：待分派
- pending_process：待处理
- processing：处理中
- pending_confirm：待用户确认
- closed：已关闭
- rejected：已驳回
- canceled：已取消

建议增加以下索引：

- `sys_user.username` 唯一索引。

- `ticket.assignee_id` 普通索引。
- `ticket.status` 普通索引。
- `ticket.category_id` 普通索引。

- `ticket_record.ticket_id` 普通索引。
- `ticket_comment.ticket_id` 普通索引。

- `operation_log.business_type, business_id` 组合索引。
