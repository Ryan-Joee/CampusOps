# AI 开发工作流

## 1. 文档目的

本文档用于记录 CampusOps 项目中人类开发者、Codex、Claude Code 以及可能使用的 Claude Code skills 之间的协作方式。

项目目标不是简单让 AI 生成代码，而是建立一套可持续使用的 AI Agent 辅助开发流程。

## 2. 角色分工

### 2.1 人类开发者

负责：

- 最终需求判断。
- 技术取舍。
- 代码审查。
- 运行和验证结果。
- 决定是否引入新工具或依赖。

### 2.2 Codex

主要负责：

- 需求分析。
- 架构讨论。
- 文档整理。
- 技术决策记录。
- 代码审查建议。
- 任务拆分。

### 2.3 Claude Code

主要负责：

- 根据明确任务修改代码。
- 初始化项目骨架。
- 实现后端或前端模块。
- 补充测试。
- 根据报错迭代修复。

Claude Code 应优先从项目根目录运行，而不是只进入后端或前端子目录。

## 3. Claude Code 使用原则

每次使用 Claude Code 前，应明确：

- 本次任务目标。
- 允许修改的目录。
- 必须阅读的文档。
- 不允许做的事情。
- 验收标准。

示例：

```text
请先阅读 CLAUDE.md、docs/00-项目总览.md、docs/04-系统架构设计.md、docs/07-权限设计.md，
然后只在 campusops-server 目录中搭建后端基础包结构、统一响应和全局异常处理。
本次不要实现具体工单业务，也不要添加 AI 依赖。
```

## 4. skills 使用原则

Claude Code skills 可以增强开发流程，但不应替代项目文档和人工判断。

### 4.1 superpower

`superpower` 可以作为候选 skill，但建议在完成一次普通 Claude Code 小任务后再决定是否安装。

如果安装，需要补充：

- 它解决什么问题。
- 它改变了哪些工作流。
- 使用它时需要遵守哪些项目边界。

### 4.2 前端 skill

前端 skill 可以考虑使用，但必须服务于 CampusOps 的工作台产品形态。

前端设计原则：

- 管理后台风格。
- 信息清晰。
- 操作高效。
- 表格、表单、详情页优先。
- 不做营销页。
- 不做过度炫酷的大屏。
- AI 面板是业务辅助，不是页面主体。

## 5. 单次任务推荐格式

每次交给 AI Agent 的任务建议包含：

```text
目标：
允许修改范围：
必须阅读文档：
禁止事项：
验收标准：
```

例如：

```text
目标：实现工单创建接口。
允许修改范围：campusops-server。
必须阅读文档：docs/01-需求说明.md、docs/03-业务流程.md、docs/06-接口设计.md、docs/07-权限设计.md。
禁止事项：不要实现 AI 模块，不要修改前端，不要引入 Redis。
验收标准：普通用户可以创建工单，创建后状态为 pending_assignment，并记录操作日志。
```

## 6. 开发节奏

推荐节奏：

1. 人类开发者和 Codex 明确需求。
2. Codex 整理文档和任务边界。
3. Claude Code 执行具体编码。
4. 人类开发者运行项目和测试。
5. Codex 或 Claude Code 协助修复问题。
6. 完成后更新文档或 ADR。

## 7. 当前阶段要求

当前阶段仍属于工程启动准备期。

在前端技术栈、数据库 SQL、后端基础骨架和 Agent 工作流明确前，不应直接进入完整业务开发。
