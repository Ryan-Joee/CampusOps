# 001-后端基础骨架

## 任务目标

在 `campusops-server/` 中建立后端基础工程骨架，为后续登录认证、用户权限、工单业务开发做准备。

本任务只做基础设施代码，不实现完整业务流程。

## 开发前必读

请先阅读：

- `CLAUDE.md`
- `docs/00-项目总览.md`
- `docs/04-系统架构设计.md`
- `docs/06-接口设计.md`
- `docs/07-权限设计.md`
- `docs/adr/0002-后端技术选型.md`

## 技术约束

- Spring Boot：`3.5.15`
- Java：`17`
- ORM：MyBatis Plus
- 不使用 JPA
- 不引入 Redis、消息队列、Spring AI、LangChain4j、Flyway、Liquibase
- 接口统一前缀：`/campusops`
- 后端运行端口：`3016`
- `application.yml` 中数据库账号密码为本地开发占位配置，可按本机 MySQL 修改，不要提交真实生产密钥

## 本次允许修改范围

只允许修改：

- `campusops-server/pom.xml`
- `campusops-server/src/main/java/com/campusops/**`
- `campusops-server/src/main/resources/application.yml`
- `campusops-server/src/test/java/com/campusops/**`

如需修改其他文件，先说明原因，不要直接扩大范围。

## 需要实现的内容

### 1. 基础包结构

建议建立以下包：

```text
com.campusops
  common
    result
    exception
    page
  config
  security
  auth
  user
  ticket
  knowledge
  ai
```

### 2. 统一响应

实现统一响应对象，例如：

```json
{
  "code": "SUCCESS",
  "message": "OK",
  "data": {},
  "traceId": null
}
```

建议类名：

- `ApiResponse`
- `ResultCode`

### 3. 全局异常处理

实现基础异常体系：

- `BusinessException`
- `GlobalExceptionHandler`

至少处理：

- 业务异常
- 参数校验异常
- 未捕获异常

### 4. 分页对象

实现基础分页响应对象：

- `PageResult<T>`

字段建议：

- `items`
- `page`
- `pageSize`
- `total`

### 5. MyBatis Plus 配置

实现基础配置类：

- `MybatisPlusConfig`

至少包含分页插件配置。

### 6. OpenAPI 配置

实现基础配置类：

- `OpenApiConfig`

用于描述项目名称、版本、说明。

Swagger UI 路径应与 `application.yml` 保持一致。

### 7. Spring Security 临时配置

第一阶段先允许以下内容：

- 放行 Swagger / OpenAPI 相关路径
- 放行后续登录接口 `/campusops/auth/login`
- 其他接口暂时要求认证或先统一放行，二选一即可，但必须在代码中保持清晰

注意：本任务不要求完整实现 JWT 登录认证。

## 明确不要做

- 不实现登录接口业务
- 不创建完整用户、角色、权限 CRUD
- 不实现工单业务
- 不接入真实 AI
- 不改 SQL 表结构
- 不写复杂 AOP 日志

## 验收标准

- 后端项目可以编译通过
- Spring Boot 应用可以启动
- `/campusops/swagger-ui.html` 可以访问 Swagger UI
- 代码结构清晰，后续可以继续实现登录认证
- 不引入本任务未允许的新技术
