# 002-登录认证与当前用户接口

## 任务目标

在 `campusops-server/` 中实现第一版登录认证能力，形成最小可用的认证闭环：

- 用户使用账号密码登录。
- 登录成功后返回 JWT Access Token。
- 前端请求受保护接口时通过 `Authorization: Bearer <token>` 访问。
- 后端可以通过 Token 识别当前用户。
- 提供当前用户信息接口。

本任务只做认证基础能力，不做用户管理 CRUD、不做角色权限管理页面、不做工单业务。

## 需要实现的接口

### 1. 登录接口

```text
POST /campusops/auth/login
```

Controller 实际映射：

```text
POST /auth/login
```

请求体：

```json
{
  "username": "student001",
  "password": "123456"
}
```

成功响应的 `data`：

```json
{
  "accessToken": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 7200
}
```

失败规则：

- 用户不存在：返回统一错误响应，不暴露“用户名不存在”的细节。
- 密码错误：返回统一错误响应，不暴露“密码错误”的细节。
- 用户状态不是 `enabled`：返回统一错误响应，提示账号已禁用。
- 参数为空：返回参数校验错误。

### 2. 当前用户接口

```text
GET /campusops/auth/me
```

Controller 实际映射：

```text
GET /auth/me
```

请求头：

```text
Authorization: Bearer <accessToken>
```

成功响应的 `data`：

```json
{
  "id": 5,
  "username": "student001",
  "realName": "张三",
  "email": "student001@campusops.local",
  "phone": "13800000005",
  "department": "计算机学院",
  "roles": ["normal_user"]
}
```

失败规则：

- 未携带 Token：返回 `UNAUTHORIZED`。
- Token 无效或过期：返回 `UNAUTHORIZED`。
- Token 对应用户不存在或已禁用：返回 `UNAUTHORIZED`。

## 密码校验要求

当前 `data.sql` 中开发账号使用 `{noop}` 密码格式，例如：

```text
student001 / 123456
admin / admin123
```

本任务应将 `SecurityConfig` 中的 `PasswordEncoder` 调整为：

```java
PasswordEncoderFactories.createDelegatingPasswordEncoder()
```

原因：

- 开发阶段兼容 `{noop}` 初始化数据。
- 后续正式用户可以使用 `{bcrypt}` 格式密码。
- 不需要为了本任务修改初始化 SQL。

## JWT 要求

### 配置

继续使用 `application.yml` 中已有配置：

```yaml
campusops:
  security:
    jwt:
      secret: change-this-dev-secret-change-this-dev-secret
      expiration-seconds: 7200
```

新增 `JwtProperties` 读取该配置。

### Token 内容

JWT 至少包含：

- `sub`：用户 ID 字符串
- `username`：用户名
- `roles`：角色编码数组
- `iat`：签发时间
- `exp`：过期时间

### Token 校验

必须校验：

- 签名合法。
- Token 未过期。
- `sub` 可以解析为用户 ID。
- 用户仍存在且状态为 `enabled`。

## Security 配置要求

修改 `SecurityConfig`：

- 保持无状态会话。
- 继续关闭 CSRF、formLogin、httpBasic。
- 放行以下路径：
  - `/auth/login`
  - `/swagger-ui.html`
  - `/swagger-ui/**`
  - `/v3/api-docs`
  - `/v3/api-docs/**`
  - `/error`
- 其他接口改为必须认证：

```java

```

- 注册 `JwtAuthenticationFilter`。
- 使用 `RestAuthenticationEntryPoint` 返回统一 `UNAUTHORIZED` 响应。
- 使用 `RestAccessDeniedHandler` 返回统一 `FORBIDDEN` 响应。

## 业务规则

### 登录流程

1. 校验 `username`、`password` 非空。
2. 根据用户名查询用户。
3. 用户不存在时抛出统一认证失败异常。
4. 用户状态不是 `enabled` 时抛出账号禁用异常。
5. 使用 `PasswordEncoder.matches(rawPassword, passwordHash)` 校验密码。
6. 查询用户角色编码。
7. 生成 JWT。
8. 返回 `LoginResponse`。

### 当前用户流程

1. 从 Spring Security 上下文获取当前认证用户。
2. 根据用户 ID 查询最新用户信息。
3. 查询角色编码。
4. 返回 `CurrentUserVO`。

## 统一错误响应要求

可在 `ResultCode` 中补充：

- `AUTH_FAILED`：用户名或密码错误。
- `ACCOUNT_DISABLED`：账号已禁用。
- `TOKEN_INVALID`：登录状态无效或已过期。

响应仍使用 `ApiResponse`。

认证失败 HTTP 状态建议：

- 登录账号密码错误：`401`
- Token 无效或过期：`401`
- 已认证但无权限：`403`
- 参数校验错误：`400`
