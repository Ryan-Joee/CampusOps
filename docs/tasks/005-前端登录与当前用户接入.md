# 005-前端登录与当前用户接入

## 任务目标

在 `campusops-front/` 中完成前端登录闭环：登录页调用后端认证接口、保存 Token、请求拦截器携带 `Authorization`、路由守卫保护工作台页面、刷新后通过 `/auth/me` 恢复当前用户信息，并提供可用的退出登录。

本任务只做登录与当前用户接入，不做注册、不做工单业务页面开发。

## 技术约束

- 本任务只允许修改 `campusops-front/**`。
- 不修改 `campusops-server/**`。
- 不修改数据库、SQL、后端任务文档。
- 不修改 `.agents/**`、`.codex/**`、`.claude/**`。
- 不新增前端依赖，优先使用现有 Vue 3、Vue Router、Pinia、Element Plus、Axios。
- 当前后端没有注册接口，不实现真实注册功能。

## 当前后端接口

后端上下文路径为 `/campusops`。

### 登录

```text
POST /campusops/auth/login
```

前端 `request` 已配置 `baseURL: '/campusops'`，因此调用路径应为：

```text
POST /auth/login
```

请求：

```json
{
  "username": "student001",
  "password": "123456"
}
```

响应 `data`：

```json
{
  "accessToken": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 7200
}
```

### 当前用户

```text
GET /campusops/auth/me
```

前端调用路径：

```text
GET /auth/me
```

请求头：

```text
Authorization: Bearer <accessToken>
```

响应 `data` 包含：

- `id`
- `username`
- `realName`
- `email`
- `phone`
- `department`
- `roles`

## 实现范围

### 1. 登录页接入真实接口

修改 `campusops-front/src/views/LoginView.vue`：

- 使用现有 `login` API 调用后端。
- 表单校验用户名和密码必填。
- 登录按钮有 loading 状态。
- 登录失败时显示清晰错误提示。
- 登录成功后保存 Token，拉取当前用户信息，然后跳转 `/dashboard`。
- 如果已经登录并且当前用户信息有效，访问 `/login` 时跳转 `/dashboard`。
- 登录页不要提供真实注册入口。

建议开发账号提示可低调显示，不要做成营销式大横幅：

- `student001 / 123456`
- `admin / admin123`
- `service_admin / admin123`

### 2. 用户状态管理

修改 `campusops-front/src/stores/user.js`：

- 保留 `token` 和 `userInfo`。
- 新增或完善：
  - `loginAction(credentials)`
  - `fetchCurrentUser()`
  - `logoutAction()`
  - `clearSession()`
- Token 继续存储在 `localStorage` 的 `campusops_token`。
- `logoutAction` 只做前端本地退出；当前后端没有 `/auth/logout`，不要调用不存在的退出接口。

### 3. 请求封装

修改 `campusops-front/src/api/request.js`：

- 请求拦截器继续从 `localStorage` 读取 Token 并设置 `Authorization: Bearer <token>`。
- 响应拦截器统一返回后端响应体。
- 当后端返回 `UNAUTHORIZED`、HTTP 401 或 Token 失效时，清理本地登录态并跳转 `/login`。
- 错误消息优先使用后端返回的 `message`。
- 注意避免在登录接口失败时造成重复跳转或死循环。

### 4. 路由守卫

修改 `campusops-front/src/router/index.js`：

- 给 `/login` 标记为公开页面。
- 给工作台主布局下页面标记为需要登录，或默认除 `/login` 外都需要登录。
- 未登录访问受保护页面时跳转 `/login`，并保留 `redirect` 查询参数。
- 已登录但 `userInfo` 为空时，调用 `fetchCurrentUser()` 恢复用户信息。
- 恢复失败时清理登录态并跳转 `/login`。

### 5. 布局当前用户和退出

修改 `campusops-front/src/layouts/BasicLayout.vue`：

- 顶部右侧显示当前用户真实姓名；没有真实姓名时显示用户名。
- 可显示部门和角色，保持克制。
- 退出登录时清理本地状态并跳转 `/login`。
- 不调用后端 logout 接口。

### 6. 修复中文文案和页面粗糙感

当前前端部分页面存在中文乱码或标签片段异常，登录闭环涉及的文件必须修复：

- `LoginView.vue`
- `BasicLayout.vue`
- `router/index.js` 中的页面标题
- 必要的 `style.css` 文案相关样式

不要求本任务修复所有业务占位页，但不应让登录页和基础布局继续出现乱码。

## 视觉与交互要求

CampusOps 是校园 IT 服务工作台，不是营销落地页。

登录页应体现“校园 IT 服务入口”的可信、清爽、实用：

- 页面第一屏就是登录，不做产品宣传长页。
- 不使用夸张渐变背景、大面积紫蓝光效、装饰性圆球、玻璃拟态卡片。
- 不堆砌“AI 赋能、智能、高效”等泛化口号。
- 不使用巨大 hero 标题挤占表单空间。
- 视觉层级保持安静：品牌、系统说明、表单、辅助账号提示即可。
- Element Plus 组件可以继续使用，但需要调整间距、文案、错误态和 loading 态，让界面像真实后台系统。
- 卡片圆角不超过 8px。
- 按钮和输入框在 1366px 桌面和 390px 移动宽度下都不能文字溢出或互相遮挡。

建议布局：

- 桌面端：左侧为简洁系统身份区，右侧为登录表单。
- 移动端：单列布局，表单优先，说明文案收敛。
- 背景使用浅灰或非常克制的浅蓝灰，不做花哨装饰。

## 明确不要做

- 不做注册页面真实提交。
- 不做忘记密码流程。
- 不做验证码。
- 不做 SSO。
- 不做工单列表、详情、创建页面的接口接入。
- 不修改后端。
- 不新增依赖。

## 验收标准

必须全部满足：

- `npm run build` 通过。
- 登录页调用 `POST /campusops/auth/login`。
- 登录成功后 Token 写入 `localStorage.campusops_token`。
- 后续请求带 `Authorization: Bearer <token>`。
- 登录成功后调用 `GET /campusops/auth/me` 并保存当前用户。
- 未登录访问 `/dashboard`、`/tickets`、`/knowledge`、`/ai-analysis` 会跳转 `/login`。
- 已登录刷新页面后能恢复当前用户信息。
- 退出登录会清理 Token 和用户信息，并跳转 `/login`。
- 登录页和基础布局没有中文乱码。
- 登录页和基础布局没有明显模板感、营销页感或 AI 生成式装饰堆叠。
