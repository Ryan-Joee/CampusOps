# 003-后端架构调整为 MVC 分层

## 任务目标

将 `campusops-server/` 后端包结构从旧版按业务域作为一级包的组织方式，调整为传统 MVC 技术分层单体结构。

本任务用于固化后端架构决策，避免后续开发继续新增 `com.campusops.auth`、`com.campusops.user`、`com.campusops.ticket`、`com.campusops.knowledge`、`com.campusops.ai` 等旧顶层业务包。

## 开发前必读

请先阅读：

- `CLAUDE.md`
- `docs/14-后端分层与包结构规范.md`
- `docs/tasks/001-后端基础骨架.md`
- `docs/tasks/002-登录认证与当前用户接口.md`

## 技术约束

- 后端实际工程目录为 `campusops-server/`，不是 `backend/`。
- Java 根包为 `com.campusops`。
- 当前阶段采用 MVC 技术分层单体架构。
- 不采用按业务域作为一级包的模块化单体结构。
- 后续业务开发必须遵守 `docs/14-后端分层与包结构规范.md`。

## 目标包结构

`campusops-server/src/main/java/com/campusops` 下一级包应按技术层组织：

```text
com.campusops
  common
  config
  controller
  dto
  entity
  mapper
  security
  service
  vo
```

允许在 `dto`、`vo` 等出入参包下按业务域继续划分子包，例如：

```text
dto.auth
vo.auth
```

但不允许将业务域作为 `com.campusops` 下的一级包。

## 验收标准

必须全部满足：

- `campusops-server/src/main/java/com/campusops` 存在。
- `backend/src/main/java/com/campusops` 不作为当前后端路径使用。
- `com.campusops` 下一级包按技术层组织。
- 不存在以下旧顶层业务包：
  - `com.campusops.auth`
  - `com.campusops.user`
  - `com.campusops.ticket`
  - `com.campusops.knowledge`
  - `com.campusops.ai`
- 不存在以旧顶层业务包声明的 Java package：
  - `package com.campusops.auth...`
  - `package com.campusops.user...`
  - `package com.campusops.ticket...`
  - `package com.campusops.knowledge...`
  - `package com.campusops.ai...`
- 后续新增代码继续遵守 `docs/14-后端分层与包结构规范.md`。

## 明确不要做

- 不要恢复旧版 `auth/user/ticket/knowledge/ai` 一级业务包。
- 不要为了单个功能新增 `module`、`manager`、`helper`、`misc` 等含义模糊的顶层包。
- 不要修改 `.agents/`、`.codex/`、`.claude/` 等本地 Agent 工具目录。
- 不要在本任务中引入新依赖或调整数据库表结构。
