# 工程启动计划

## 1. 文档目的

本文档用于规划 CampusOps 正式进入编码前需要完成的准备工作，避免在前端技术栈、AI Agent 工作流、数据库初始化和后端依赖尚未确定时过早进入业务开发。

当前阶段的目标不是马上写业务代码，而是把项目启动路线定清楚。

## 2. 当前项目状态

已完成：

- 项目方向确定为 CampusOps。
- 已创建 `docs/` 文档目录。
- 已创建 `campusops-server/` 后端目录。
- 已创建 `campusops-front/` 前端目录。
- 后端工程已由 IDEA 创建。
- 后端版本为 Spring Boot 3.5.15。
- Java 版本为 17。
- ORM 已决定使用 MyBatis Plus，不使用 JPA。
- 已记录后端技术选型 ADR。

尚未完成：

- 前端技术栈选择。
- Claude Code / Codex 协作流程细化。
- 是否安装 Claude Code skill，例如 superpower。
- 是否使用前端相关 skill。
- 数据库初始化 SQL。
- 后端基础骨架设计。
- 前端页面结构规划。

## 3. 启动阶段总原则

- 不急于写业务代码。
- 不提前引入不熟悉且非必要的工具。
- 不为了长期扩展过早增加复杂度。
- 每次引入新技术都要说明原因和使用边界。
- 文档先约束方向，代码再按文档实现。
- AI Agent 可以辅助开发，但必须遵守项目文档和目录边界。

## 4. 推荐启动顺序

### 4.1 确定 AI Agent 协作方式

需要明确：

- Claude Code 从项目根目录运行。
- Claude Code 开发前必须阅读 `CLAUDE.md` 和相关 `docs/` 文档。
- Codex 主要用于规划、审查、补文档和设计决策。
- Claude Code 主要用于执行具体编码任务。
- 每次任务应限制修改范围，例如只改 `campusops-server/` 或只改 `campusops-front/`。

### 4.2 评估 Claude Code skills

可以考虑安装 `superpower`，但不要在未理解其工作方式前强依赖它。

建议顺序：

1. 先用普通 Claude Code 完成一个小任务。
2. 观察它是否能遵守 `CLAUDE.md` 和项目文档。
3. 再决定是否安装 `superpower`。
4. 如果安装，将使用方式写入 Agent 协作说明。

前端 skill 也应按需引入，不应让 skill 决定产品形态。

### 4.3 确定前端技术栈

前端选型需要单独讨论和记录 ADR。

前端应服务于工单工作台场景，重点是：

- 表格。
- 表单。
- 详情页。
- 状态标签。
- 权限菜单。
- 右侧 AI 辅助面板。
- 管理后台布局。

不应优先追求营销页、炫酷大屏或复杂动画。

### 4.4 确定数据库初始化策略

第一阶段不引入 Flyway 或 Liquibase。

采用简单直观的 SQL 文件方式：

```text
campusops-server/src/main/resources/sql/schema.sql
campusops-server/src/main/resources/sql/data.sql
```

其中：

- `schema.sql` 用于创建表结构。
- `data.sql` 用于初始化角色、用户、分类和示例工单。

后续如果项目规模扩大，再考虑数据库迁移工具。

### 4.5 确定后端依赖添加顺序

当前 `pom.xml` 已包含：

- Spring Web
- Spring Security
- Validation
- MySQL Driver
- Lombok
- Spring Boot Test
- Spring Security Test

下一阶段可以添加：

- MyBatis Plus
- Springdoc OpenAPI
- JWT 相关依赖

暂不添加：

- Redis
- Spring AI
- LangChain4j
- RabbitMQ / Kafka
- 向量数据库客户端
- 工作流引擎

### 4.6 规划后端基础骨架

后端第一步只搭基础骨架，不写复杂业务。

基础骨架包括：

- 包结构。
- 统一响应格式。
- 全局异常处理。
- 参数校验错误处理。
- 基础安全配置。
- MyBatis Plus 配置。
- OpenAPI 配置。
- `application.yml` 与开发环境配置。

### 4.7 规划前端基础骨架

前端技术栈确定后，再规划：

- 路由结构。
- 布局结构。
- 登录页。
- 工单列表页。
- 工单详情页。
- 知识库页。
- 管理员工作台。
- AI 辅助面板组件。

## 5. 第一阶段开发顺序建议

正式编码后，推荐顺序：

1. 后端基础骨架。
2. 数据库 SQL 初始化。
3. 用户、角色和登录。
4. 工单创建和查询。
5. 工单分派和处理。
6. 工单关闭和评价。
7. 知识库基础能力。
8. 简单统计。
9. AI Mock 能力。
10. 前端核心页面串联。

AI 真正接入 LLM 应放在传统工单闭环稳定之后。

## 6. 当前明确不做

当前阶段不做：

- 复杂数据库迁移工具。
- 多 Agent 架构。
- MCP 接入。
- RAG 知识库。
- Redis 缓存。
- 消息队列。
- 大屏可视化。
- 复杂工作流引擎。

这些内容保留为后续演进方向。
