# ADR 0001：项目方向决策

## 状态

已接受

## 背景

本项目是本科毕业设计，同时也是一次使用 AI Agent 辅助软件开发的长期实践。

项目目标不是完成一个纯 AI Demo，而是构建一个以传统 Java 后端业务为主体、AI 作为增强能力的软件系统。

候选方向包括 AI 工单系统、AI 项目协作平台、AI 设备资产运维平台等。

## 决策

选择 CampusOps，即“面向校园 IT 服务的 AI 增强型工单管理平台”作为毕业设计方向。

## 原因

选择该方向的主要原因：

- 校园 IT 服务问题具有真实业务场景，容易理解和演示。
- 工单系统具备完整业务闭环，包括提交、分派、处理、确认和归档。
- Java 后端可以承担核心业务能力，包括权限、状态流转、数据库设计、日志审计和统计分析。
- AI 接入点自然，包括分类、摘要、相似工单推荐和知识库推荐。
- 系统能够持续沉淀历史工单和知识库文章，具备长期演进价值。
- 后续可以平滑扩展到 RAG、工具调用、Agent、MCP 和多 Agent 协作。

## 影响

后续需求、架构和开发工作都围绕校园 IT 服务工单场景展开。

第一阶段优先保证传统工单系统可用，AI 能力作为增强模块逐步接入。

系统设计必须保留长期演进空间，但不能让第一版范围失控。
