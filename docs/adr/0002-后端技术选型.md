# ADR 0002：后端技术选型

## 状态

已接受

## 背景

CampusOps 后端需要支撑校园 IT 工单处理、用户权限、状态流转、知识库、统计分析、AI 辅助能力和后续 RAG / Agent 演进。

项目已经通过 IDEA 创建 Spring Boot 工程，后端目录为 `campusops-server/`。

## 决策

后端第一阶段采用以下技术选型：

- Java 17
- Spring Boot 3.5.15
- Maven
- Spring Web
- Spring Security
- Spring Validation
- MySQL Driver
- Lombok
- MyBatis Plus
- JUnit 5
- Spring Security Test

项目不使用 JPA。

## 原因

选择 Spring Boot 3.5.15 和 Java 17，是为了使用较新的 Spring Boot 3 生态，同时保持 Java 版本兼容性和稳定性。

选择 MyBatis Plus，而不是 JPA，原因如下：

- 工单列表存在较多动态筛选条件，例如状态、分类、优先级、提交人、处理人、时间和关键词。
- 管理端统计需要较多聚合查询，例如分类占比、平均处理时长、处理人工作量和超时数量。
- 数据权限需要较强 SQL 可控性，例如普通用户只能看自己的工单，处理人只能看分配给自己的工单。
- 后续 AI 推荐、知识库搜索、相似工单检索会需要灵活查询能力。
- MyBatis Plus 能兼顾常规 CRUD 效率和复杂 SQL 可控性。

选择 Spring Security，是为了提供认证、接口级权限和测试支持。业务数据权限仍需要在 Service 层实现。

选择 Spring Validation，是为了在 DTO 层统一处理请求参数校验。

## Maven 依赖约束

后续添加 Maven 依赖时必须遵守：

- Spring Boot 已管理版本的依赖，不手动指定版本。
- 第三方依赖必须确认兼容 Spring Boot 3.5.15 和 Java 17。
- 不因为未来规划提前添加未使用依赖。
- AI、Redis、消息队列、向量数据库等依赖应在对应模块进入实现阶段后再添加。
- MyBatis Plus 版本需要选择支持 Spring Boot 3 的 starter。

## 第一阶段暂不引入

第一阶段暂不引入：

- JPA
- Redis
- Spring AI
- LangChain4j
- RabbitMQ / Kafka
- 向量数据库客户端
- 复杂工作流引擎

这些技术可以在后续 ADR 中单独评估。

## 影响

后端代码应以 MyBatis Plus 的 Mapper、Service 和分页能力为基础，不引入 JPA Entity Repository 模式。

权限设计采用 Spring Security + RBAC + Service 层数据权限的组合。

AI 模块后续应作为独立能力模块接入，不应散落在 Controller 中。
