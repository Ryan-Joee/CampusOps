# CampusOps：面向校园 IT 服务的 AI 增强型工单管理平台 - 项目总览

## 1. 项目名称

CampusOps，中文名为“面向校园 IT 服务的 AI 增强型工单管理平台”。

## 2. 项目定位

本项目面向高校校园 IT 服务场景，建设一个以传统 Java 后端业务为主体、以 AI 能力为增强模块的工单管理平台。

系统用于统一接收、分派、处理和沉淀校园 IT 服务问题，例如校园网异常、统一身份认证失败、教务系统无法访问、VPN 连接失败、软件环境安装问题、机房设备故障等。

AI 在系统中不作为唯一主体，也不直接替代人工决策，而是用于辅助工单分类、问题摘要、相似案例推荐、知识库推荐、回复草稿生成和服务运营分析。

## 3. 建设背景

校园 IT 服务问题通常具有以下特点：

- 问题来源分散，常见于微信群、电话、邮件或线下反馈。
- 处理过程缺少统一记录，问题进度不透明。
- 同类问题反复出现，历史经验难以复用。
- 管理人员难以及时掌握高频问题、超时问题和服务质量。
- 用户描述多为自然语言，适合引入 AI 进行理解、摘要和推荐。

因此，本项目希望通过工单系统建立稳定的业务闭环，并在此基础上逐步引入 AI 能力，提高服务处理效率和知识沉淀能力。

## 4. 核心目标

第一阶段目标是实现一个完整可用的校园 IT 工单管理系统，包含用户提交、管理员分派、处理人处理、用户确认、知识库沉淀和基础统计能力。

第二阶段目标是在传统业务系统基础上接入 AI 辅助能力，包括工单自动分类、工单摘要、相似工单推荐和知识库推荐。

长期目标是将系统演进为支持 RAG、工具调用、Agent 工作流、MCP 接入和多 Agent 协作的 AI Native 服务运营平台。

## 5. 项目原则

- 传统业务系统是主体，AI 是增强能力。
- AI 只提供建议，不直接执行高风险业务操作。
- 权限、状态流转、数据一致性、审计日志必须由后端系统保证。
- 系统在没有 AI 的情况下仍应保持核心业务可用。
- 所有 AI 能力都应具备调用记录、失败降级和人工确认边界。
- 文档不仅服务于毕业设计，也服务于 AI Agent 辅助开发。

## 6. 第一版范围

第一版 MVP 重点完成以下能力：

- 用户登录与基础角色权限。
- 普通用户提交工单、查看进度、补充信息、确认关闭。
- 服务管理员查看工单、分派处理人、调整分类和优先级。
- IT 处理人接单、回复、更新状态、填写处理记录、标记解决。
- 知识库文章的创建、查询和与工单关联。
- 工单状态变更记录和操作日志。
- AI 辅助工单分类、摘要和相似内容推荐的基础实现。
- 工单数量、分类占比、处理时长、超时数量等基础统计。

## 7. 第一版不做什么

第一版暂不实现以下内容：

- 复杂低代码流程引擎。
- 多租户企业级 ITSM 平台。
- 完整多 Agent 自动处理系统。
- AI 自动关闭工单或自动修改关键业务状态。
- 大规模向量数据库集群和复杂知识图谱。
- 高复杂度前端可视化大屏。
- 与真实校园统一认证、监控平台、邮件系统的深度集成。

这些内容可以作为毕业后的长期演进方向。

## 8. 初步技术栈

后端技术栈已初步确定：

- Java 17
- Spring Boot 3.5.15
- Spring Security
- MyBatis Plus
- MySQL
- Redis
- OpenAPI / Swagger
- JUnit 5
- Docker Compose
- Spring AI 或 LangChain4j

其中 Redis、OpenAPI、Docker Compose、Spring AI 或 LangChain4j 属于后续按阶段引入的能力，不在后端工程初始化时一次性加入。

前端技术栈可后续确定，优先选择适合快速构建管理后台和工作台的方案。

## 9. 长期演进路线

本项目后续可以按以下阶段演进：

1. 传统工单系统：完成工单闭环、权限、知识库、统计和日志。
2. AI 辅助能力：接入 LLM，实现分类、摘要、推荐和回复草稿。
3. RAG 知识库：对知识库文章和历史工单进行检索增强。
4. 工具调用：让 AI 在用户确认后调用后端工具查询、创建草稿或生成报告。
5. Agent 工作流：构建分类 Agent、诊断 Agent、知识维护 Agent 等单一职责能力。
6. MCP 接入：连接外部日志、监控、文档、通知等工具。
7. 多 Agent 协作：实现面向服务运营的多角色智能协同。

## 10. 毕业设计价值

本项目可以体现以下能力：

- 传统 Java 后端业务建模能力。
- 用户、角色、权限、状态流转和数据设计能力。
- 缓存、异步任务、日志审计、接口设计和测试能力。
- AI 与传统业务系统结合的工程边界设计能力。
- AI Agent 辅助软件开发过程中的文档化、任务化和评测化实践能力。
